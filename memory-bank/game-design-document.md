# 凡人修仙传：掌上仙途 - 游戏设计文档 (GDD)

## 1. 游戏概述 (Game Overview)
*   **游戏名称**：凡人修仙传：掌上仙途
*   **游戏类型**：放置挂机 / 文字修真 / 角色扮演 (H5 移动端网页游戏)
*   **核心体验**：还原韩立从凡人到道祖的逆袭之路，强调“资源管理”、“谨慎探索”与“机缘获取”。
*   **目标受众**：修仙小说爱好者、碎片化时间玩家。

## 2. 世界观与背景 (Setting)
基于《凡人修仙传》原著设定。
*   **修仙境界**：炼气、筑基、结丹、元婴、化神、炼虚、合体、大乘、渡劫。
*   **灵根设定**：伪灵根（主角开局）、真灵根、天灵根、变异灵根。灵根决定修炼速度。
*   **势力**：七玄门（新手村）、黄枫谷、乱星海、天南大陆等。

## 3. 核心玩法 (Core Gameplay)

### 3.1 挂机修炼 (Idle Cultivation)
*   **打坐**：随时间自动获取修为（真气）。
*   **瓶颈与突破**：达到境界圆满需服用特定丹药（如筑基丹）进行突破，有失败几率，失败可能导致修为倒退。
*   **神秘小瓶（金手指）**：
    *   玩家每日可获得“绿液”。
    *   绿液用于催熟灵药，将普通草药变为千年灵药，是炼制极品丹药的核心来源。

### 3.2 探索与战斗 (Exploration & Combat)
*   **副本探索**：文字MUD形式结合Roguelike元素。
    *   *事件*：遭遇妖兽、发现古修洞府、偶遇杀人夺宝。
    *   *策略*：遇到强敌可选择“韩跑跑”战术（逃跑），保留性命。
*   **战斗系统**：
    *   回合制自动战斗。
    *   **法宝与符箓**：战斗前配置，关键时刻自动释放。符箓为消耗品，威力大；法宝需温养。

### 3.3 经营与养成 (Management & Progression)
*   **洞府经营**：
    *   *药园*：种植灵草，使用绿液催熟。
    *   *炼丹房*：根据丹方炼制提升修为或突破瓶颈的丹药。
    *   *炼器室*：打造法器、法宝。
*   **坊市交易**：
    *   购买丹方、种子、基础材料。
    *   出售多余法宝、低级丹药换取灵石。

## 4. 游戏系统 (Game Systems)

| 系统名称 | 描述 | 凡人特色 |
| :--- | :--- | :--- |
| **灵根系统** | 决定基础修炼速度 | 开局随机，四/五灵根（伪灵根）需靠丹药堆上去。 |
| **功法系统** | 提升属性与特定技能 | 如《青元剑诀》、《大衍决》（提升神识）。 |
| **神识系统** | 影响探索范围与出手速度 | 凡人特色，神识强可先手或发现隐形敌人。 |
| **机缘系统** | 随机事件 | 捡漏、由于心性谨慎避开陷阱等。 |

## 5. 界面与交互 (UI/UX)
*   **风格**：水墨国风，简约文字风，强调修仙的清冷与神秘感。
*   **布局**：
    *   *底部导航*：修炼（主页）、洞府（生产）、历练（战斗）、行囊（背包）、坊市（商店）。
    *   *主界面*：显示角色境界、当前修炼速度、当前所在地图。

## 6. 技术架构 (Technical Stack)
*   **前端**：HTML5, Vue.js / React (适合移动端Web), Canvas (简单特效)。
*   **后端**：Node.js / Python, WebSocket (实时战斗/聊天)。
*   **数据存储**：MySQL (玩家数据), Redis (缓存排行榜/状态)。

## 7. 变现模式 (Monetization)
*   **广告**：观看广告获得额外的“绿液”滴数或加速修炼。
*   **内购**：购买外观（皮肤）、改名卡、扩充背包（不直接售卖修为，保持数值公平）。
